## 1. AWSビリング：確認すべきページと頻度

### 毎月1日に確認（3ページ、計10分）

#### ① クレジット残高（最重要）

```
AWSコンソール → 請求とコスト管理 → クレジット
```

| 確認項目 | 目安 |
|---|---|
| 合計残高の変化 | 先月比で想定どおりか |
| 各クレジットの使用額 | 「AWS Free Tier」が主に減っているはず |
| 有効期限 | 2027/04/24まで有効 |

**アラート基準:**
- 残高 $80 以下 → 移行計画を本格始動
- 残高 $30 以下 → 緊急移行

---

#### ② Cost Explorer（サービス別内訳）

```
AWSコンソール → コストと使用状況の分析 → Cost Explorer
→ グループ化: サービス
→ 期間: 先月
```

確認項目:
- ALB / ECS / RDS / Secrets Manager それぞれいくらか
- 前月比で急増しているサービスはないか
- 無料枠が尽きて急に課金が始まっていないか

---

#### ③ 無料利用枠の使用状況

```
AWSコンソール → 請求とコスト管理 → 無料利用枠
```

確認項目:
- 80%以上使用しているサービスがあれば超過リスクあり
- 2027/04/24に期限が切れるサービス一覧を把握しておく

> **注意:** RDS db.t4g.micro が無料枠に含まれているか（t4g がリストにあるか）をここで確認する。  
> 含まれていなければ月 $11〜14 が別途かかっている可能性あり。

---

### アラートが来たら即確認

```
設定済み: $30超で 9p96d9@gmail.com に通知
```

アラートメールが来たら → Cost Explorer で原因サービスを特定。

---

### 四半期に1回（3・6・9・12月）

クレジット残高 ÷ 月間消費額 = 残り月数を計算し直す。  
計算値が変わっていたら意思決定タイムラインを更新する。

## 5. ドメイン取得戦略

### なぜ独自ドメインが必要か

- 本格発表にあたり `gto-alb-xxx.ap-northeast-1.elb.amazonaws.com` は使えない
- Firebase Auth の `authorized_domains` に追加するためのドメインが必要
- HTTPS化にはドメインが前提

---

### レジストラ比較

| レジストラ | .com年額 | .app年額 | .jp年額 | WHOIS保護 | 特徴 |
|---|---|---|---|---|---|
| **Cloudflare Registrar** | $9.77（¥1,514） | $10.13（¥1,570） | 非対応 | 無料 | 定価販売・Tunnel連携が最強 |
| **Xserver Domain** | ¥1,298 | ¥2,178 | ¥3,278 | 有料 | 日本語サポート |
| **お名前.com** | ¥1,480（初年¥1〜） | ¥1,980 | ¥2,980 | 有料 | 国内最大手・初年キャンペーン |
| **AWS Route 53** | $12（¥1,860） | $14（¥2,170） | 非対応 | 一部無料 | AWS統合・別途ホストゾーン$0.50/月 |
| **Namecheap** | $9.98（¥1,547） | $5.88（¥911） | 非対応 | 無料 | 英語圏定番・.appが安い |

---

### 状況別推奨

#### 推奨①: Cloudflare Registrar（最もシナジーが高い）

**対象:** ALB廃止後に Cloudflare Tunnel で HTTPS を実現する場合

```
ドメイン取得 → Cloudflare（レジストラ）
DNS管理     → Cloudflare（自動）
HTTPS証明書 → Cloudflare（自動・無料）
トンネル    → Cloudflare Tunnel（ALB代替・無料）
```

これにより ALB の代替が全て Cloudflare で完結し、追加費用ゼロ。  
→ **ALB廃止計画と相性が最もよい選択肢。**

**登録先:** https://www.cloudflare.com/products/registrar/

---

#### 推奨②: AWS Route 53（AWS完結にこだわる場合）

ALB に直接 A レコードで紐付けできる。設定がコンソール内で完結。  
ただしホストゾーン $0.50/月（¥78/月）が追加でかかる。  
Cloudflare Tunnel を使わないなら許容範囲。

**登録先:** AWSコンソール → Route 53 → ドメイン登録

---

#### 推奨③: お名前.com（日本語サポート重視の場合）

初年度キャンペーンで .com が格安になることがある。  
ただし DNS サーバーを Cloudflare に向ける設定（NSレコード変更）が必要になる場合がある。

---

### .com か .jp か .app か？

| TLD | 年額目安 | 向いているケース |
|---|---|---|
| .com | ¥1,300〜1,900 | 英語圏含む・汎用・信頼感 |
| .app | ¥1,500〜2,200 | アプリであることが一目でわかる。Googleがスポンサー。HTTPS必須 |
| .jp | ¥2,980〜3,300 | 日本向けサービスの明示。取得条件あり（日本の住所） |

**このアプリへの推奨:** `.app` または `.com`  
`.app` は強制的に HTTPS 必須（HSTS preload）なので、Cloudflare と組み合わせると自然にセキュアな構成になる。

---

### ドメイン取得後にやること

```
1. Cloudflare にドメインを追加（DNS管理をCloudflareに委任）
2. Firebase Console → Authentication → Authorized domains
   → 取得ドメインを追加（例: yourapp.app）
3. 既存の ALB DNS名（gto-alb-xxx.elb.amazonaws.com）を削除
4. HTTPS証明書は Cloudflare が自動発行
```
